About

I sit at the intersection of design and code.

I've watched too many good designs die in handoff — a beautiful Figma file that turns into a slow, broken site because the person coding it never understood the intent. I skip that gap entirely by doing both halves myself.

Every project starts on paper: who's looking at this, what's the one thing they need to do, and what should the page feel like in the first three seconds. The code comes after the decisions, not instead of them.
